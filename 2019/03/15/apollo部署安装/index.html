apollo部署安装

發表於2019-03-15|更新於2023-06-24|apollo
|閱讀量:

准备工作

运行环镜

  os

  建议centos7

  java
 
apollo服务端 1.8+ apollo客户端 1.7+
建议安装Java 1.8+

mysql

  Apollo的表结构对timestamp使用了多个default声明，所以需要5.6.5以上版本

环境

分布式部署需要事先确定部署的环境以及部署方式

  • DEV

    • 开发环境

  • UAT

    • 集成环境，相当于beta环境（回归测试）

  • PRO

    • 生产环境

  • Portal部署在生产环境的机房，通过它来直接管理UAT、PRO等环境的配置

  • Meta Server、Config Service和Admin Service在每个环境都单独部署，使用独立的数据库

  • Meta Server、Config Service和Admin Service在生产环境部署在两个机房，实现双活

  • Meta Server和Config Service部署在同一个JVM进程内，Admin Service部署在同一台服务器的另一个JVM进程内

注：

如果Apollo部署在公有云上，本地开发环境无法连接，但又需要做开发测试的话

  * 客户端可以升级到0.11.0版本及以上，然后通Dapollo.configService=http://config-service 的公网IP:端口来跳过meta service的服务发现

  * 可以在公有云上配置mysql，configservice与adminservice，dev环境会去数据库拉取配置

网络策略

分布式部署的时候，apollo-configservice和apollo-adminservice需要把自己的IP和端口注册到Meta Server（apollo-configservice本身）

Apollo客户端和Portal会从Meta Server获取服务的地址（IP+端口），然后通过服务地址直接访问

如果实际部署的机器有多块网卡（如docker），或者存在某些网卡的IP是Apollo客户端和Portal无法访问的（如网络安全限制），记得加上”–net=host”参数（docker跳过配置容器的独立网络栈）

eureka连接配置

需要注意的是每个环境只填入自己环境的eureka服务地址，比如UAT的apollo-configservice是3.3.3.3:8080和4.4.4.4:8080，PRO的apollo-configservice是5.5.5.5:8080和6.6.6.6:8080，那么：

1. 在UAT环境的ApolloConfigDB.ServerConfig表中设置eureka.service.url为：

http://3.3.3.3:8080/eureka/,http://4.4.4.4:8080/eureka/

2. 在PRO环境的ApolloConfigDB.ServerConfig表中设置eureka.service.url为：

http://5.5.5.5:8080/eureka/,http://6.6.6.6:8080/eureka/
