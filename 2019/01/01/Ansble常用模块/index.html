Ansble常用模块

發表於2019-01-01|更新於2023-06-24|ansible
|閱讀量:

ansible查看帮助方法

ansible-doc -l        — 查看所有模块说明信息

ansible-doc -s copy   — 表示指定查看某个模块参数用法信息

ansible kyd -m ping -vvvv  — 利用-v显示查看ansible命令执行过程

command(常用模块)

缺点：运行的命令中无法使用变量，管道。如果需要使用管道、变量，使用shell模块参数说明

chdir       —在执行一个命令前，进入一个目录中

creates     —定义一个文件是否存在，如果不存在运行相应命令，如果存在跳过此步骤

removes     —定义一个文件是否则存在，如果存在运行相应命令，如果不存在跳过此步骤

free_form*  —定义使用command模块是，-a后面必须接上linux命令命令

ansible kyd -a “hostname” == ansible kyd -m command -a “hostname”

copy(复制模块)

参数说明

src           — 推送数据的源文件信息

dest          — 推送数据的目标路径或文件信息

remote_src    — 如果为False，将搜索源信息在本地/管理及机器上，如果为True，将到远程/目标主机的机器上搜索，默认为false，目前remote_src不支持递归拷贝

backup        — 对推送传输过去的文件，进行备份

content       — 直接批量在被管理端文件中添加内容

group         — 将本地文件推送到远端，指定文件属组信息

owner         — 将本地文件推送到远端，指定文件属主信息

mode          — 将本地文件推送到远端，指定文件权限信息

命令

shell(万能模块)

优点:执行的命令中有管道或者变量，就需要使用shell

命令

script(脚本模块)

优点：此模块可以本地执行脚本文件，相当于在远程执行一样，不用再将脚本文件进行推送

命令

yum(下载安装模块)

参数说明

name      — 指定要安装的软件包名称,多个软件包用’,’隔开

state     — installed安装软件
—removed 卸载软件

命令

​

service(管理服务模块)

参数说明

name      — 定义要启动服务的名称

state     — 指定服务状态是停止或是运行，停止和运行指令要写成过去时started/stopped/restarted

enabled   — 是否让服务开启自启动

命令

file(文件操作模块)

参数说明

path            — 指定远程主机目录或文件信息

state           — directory  在远程创建目录              
— touch 创建文件                            
   — link或hard表示创建链接文件                    
— absent表示删除文件或目录

mode            — 设置文件或目录权限

owner           — 设置文件或目录属主信息

group           — 设置文件或目录属组信息

命令

cron(定时任务模块)

格式

ansible kyd -m cron -a “minute=* hour=* day=* month=* weekday=*  job=’/bin/sh /server/scripts/test.sh’”

命令

group(用户组模块)

命令

user(用户模块)

参数说明

name              —指定用户名

uid               —指定uid

groups            —指定属组

state             —创建还是删除present/absent

remover           —当state=absent时，remove=yes则表示删除用户所有信息等同于userdel -r

createhome        —是否创建家目录 yes|no

shell             —命令编辑器

命令

group(管理组模块)

name              —指定用户组

gid               —指定gid

state             —创建还是删除present/absent

system            — yes，创建系统组

命令

ping(测试模块)

测试指定主机是否能连接，如果成功返回pong命令

命令

mount(挂载模块)

参数说明

name      —挂载点

src       —挂载目录

fstype    —文件系统类型

mounted   —自动创建挂载点并挂载

state     —状态

命令

synchronize(同步模块)

参数说明

delete=yes      —使两边的内容一样（即以推送方为主）

compress=yes    —开启压缩，默认为开启

–exclude=.git  —排除 忽略同步.git结尾的文件

mode=pull       —更改推送模式为拉取模式
