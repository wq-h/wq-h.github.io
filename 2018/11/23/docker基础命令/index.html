容器生命周期管理

  • docker run命令

用于基于特定的镜像创建一个容器，并启动它

  docker run [OPTIONS] IMAGE [COMMAND] [ARG…]

  -i           表示使用交互模式，始终保持输入流开放；

  -t           表示分配一个伪终端，一般两个参数结合是使用，即可在容器中利用打开的伪终端进行交互操作

  –name  表示可以指定docker run命令启动的容器的名字，若无配置，docker会随机分配一个名字

  -p          表示用于将容器的端口暴露给宿主机的端口，通过暴露端口可以实现外部机器的访问。格式：-p hostport:container-port

  -v          表示用于挂载一个volume，可以用多个-v参数同时挂载多个volume。格式：[host:dir]:[container:dir]:[rw|ro]

  –link     添加链接到另一个容器

  • docker start/stop/restart命令

启动、停止、重启一个或多个指定容器

  -i 开启交互模式，始终保持输入流开放

  -a 附加标准输出，输出或错误输出

镜像管理

  • docker images

用于列出主机的上的镜像，使用-a 参数显示所有镜像

  docker images [OPTIONS] [REPOSITORY[:TAG]]

例：

  • docker rmi和docker rm命令

docker rmi用于删除镜像，docker rm用于删除容器。它们可同时删除多个镜像或容器

  docker rmi [OPTIONS] IMAGE [IMAGE…]

  docker rm [OPTIONS] CONTAINER [COMTAINER…]

  注意：

  使用rmi命令删除镜像时，如果有基于该镜像启动的容器存在，则无法直接删除，需要首先删除容器。当然加-f可以强制删除

容器运维操作

  • docker inspect命令

可以查看镜像和容器的详细信息，默认会列出全部信息。

  docker inspect [OPTIONS] CONTAINER|IMAGE [CONTAINER|IMAGE…]

  –format参数来指定输出的模板格式，以便输出特定信息

查看pid

  • docker ps命令

用于查看容器的相关信息

  docker ps [OPTIONS]

  -a 显示所有容器，包括未运行的

  -n 列出最近创建的n个容器

  -l 列出最新创建的容器，包括不在运行的

  • docker commit命令

用于容器创建新的镜像

  docker commit [OPTIONS] CONTAINER [REPOSITORY[:TAG]]

  -a 提交的镜像作者

  -c 使用Dockerfile指令来创建镜像

  -m 提交时的说明文字

  -p 在commit时，将容器暂停

  • docker tag命令

标记本地镜像，将其归入某一仓库

  docker tag [OPTIONS] IMAGE[:TAG] [REGISTRYHOST/][USERNAME/]NAME[:TAG]

将指定镜像保存成tar存档文件（导出）

  -o 归档的文件输出到目录

  docker load [OPTIONS]
